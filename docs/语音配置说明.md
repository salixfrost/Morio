# 语音配置说明

## 语音包选择

系统会自动选择最佳的中文语音包，优先级如下：

### Windows 系统
1. Microsoft Huihui（微软慧慧 - 女声）
2. Microsoft Kangkang（微软康康 - 男声）

### macOS 系统
1. Ting-Ting（婷婷 - 女声）
2. Sin-ji（欣姬 - 女声）
3. Mei-Jia（美佳 - 女声）

### 其他系统
- Google 普通话
- 系统默认中文语音

## 查看可用语音包

打开浏览器控制台，输入：

```javascript
voiceHelper.list()
```

会显示所有可用的语音包列表，包括：
- 索引编号
- 语音名称
- 语言代码
- 是否本地语音
- 是否默认语音

## 测试语音包

## 手动指定语音包

如果你想使用特定的语音包，可以修改代码：

### 方法1：修改 speech.js

找到 `getRecommendedChineseVoice()` 函数，修改优先级列表：

```javascript
const priorities = [
  'Your Preferred Voice Name',  // 添加你喜欢的语音名称
  'Microsoft Huihui',
  // ...
];
```

### 方法2：直接指定索引

在 `speech.js` 中，找到语音设置部分：

```javascript
const voices = synth.getVoices();
utterance.voice = voices[1];  // 使用索引 1 的语音
```

## 语音参数调整

可以调整以下参数来改变语音效果：

```javascript
utterance.rate = 1.0;    // 语速 (0.1 - 10)
                         // 1.0 = 正常
                         // 0.5 = 慢速
                         // 2.0 = 快速

utterance.pitch = 1.0;   // 音调 (0 - 2)
                         // 1.0 = 正常
                         // 0.5 = 低沉
                         // 2.0 = 尖锐

utterance.volume = 1.0;  // 音量 (0 - 1)
                         // 1.0 = 最大
                         // 0.5 = 一半
```

## 常见问题

### Q: 为什么没有声音？

A: 检查以下几点：
1. 浏览器是否支持语音合成
2. 系统音量是否打开
3. 浏览器是否有语音权限
4. 控制台是否有错误信息

### Q: 语音听起来很机械？

A: 不同浏览器和系统的语音质量不同：
- Chrome/Edge：通常质量较好
- Firefox：质量一般
- Safari：macOS 上质量最好

### Q: 如何添加更多语音包？

A: 
- Windows：设置 → 时间和语言 → 语音 → 添加语音
- macOS：系统偏好设置 → 辅助功能 → 语音内容
- Linux：安装 espeak 或其他 TTS 引擎

### Q: 可以使用自定义语音吗？

A: 浏览器 TTS API 只能使用系统安装的语音包。如需自定义语音，需要：
1. 使用第三方 TTS 服务（如 Azure、Google Cloud）
2. 使用 Web Audio API 播放预录音频
3. 使用 AI 语音合成服务

## 高级配置

### 创建自定义语音合成器

```javascript
import { createSpeechSynthesizer } from './voice-helper.js';

const synthesizer = createSpeechSynthesizer({
  lang: 'zh-CN',
  rate: 1.2,      // 稍快
  pitch: 1.1,     // 稍高
  volume: 0.9,    // 稍小
  voiceIndex: 1   // 使用索引 1 的语音
});

// 使用
await synthesizer.speak('你好，世界');

// 停止
synthesizer.stop();
```

## 调试技巧

1. 打开控制台查看语音选择日志
2. 使用 `voiceHelper` 工具测试不同语音
3. 检查 `speechSynthesis.speaking` 状态
4. 监听 `voiceschanged` 事件

## 性能优化

1. 语音列表只加载一次
2. 使用 `{ once: true }` 避免重复监听
3. 长文本分段朗读
4. 避免频繁切换语音包
